      Sysdig Agent Release Notes

      Warning

      It is highly recommended to follow upgrade best practices:

      • Keep upgrades current

      • Upgrade progressively without skipping versions, and

      • Test upgrades in a non-mission-critical or staging environment before rolling into production.

      0.91 July 17, 2019
      New Features

      Improved security Removed obsolete and vulnerable Python 2.6-compatible libraries from Docker images.

      More efficient Kubernetes event handling.

      The agent has added functionality to allow more efficient processing of Kubernetes user events.

      Reduced CPU usage on Kubernetes clusters Extended performance optimizations for processing Kubernetes Services, which will reduce agent CPU usage in large clusters.

      Container filtering enhanced. Smart filters and aggregated filtering options are now available. See Prioritize/Include/Exclude Designated Containers.

      Fixed Issues
      Monitor

      • Fixed issue with Prometheus metrics gathering intervals

        The agent will now respect the configured interval for scraping Prometheus metrics from remote endpoints, as opposed to doing it every second.

      • Fixed limit/requests calculations for init containers

        Fixed memory calculations for Kubernetes init container limits and requests

      • Improved Healthcheck monitoringAgent has improved ability to detect commands identified as a part of Kubernetes Liveness/Readiness Probes, in addition to Docker Health Checks.

      • Improved error messaging

        Warning messages for container group inconsistencies were demoted to debug level, as they are harmless and do not need to clutter the error reporting stream.

      • Fixed issue with container "incomplete" reporting status

        Starting with version 0.90.0, the agent would report containers for which it had not yet fetched metadata as "incomplete." This would then propagate to the Monitor UI. This restores the behavior where the agent leaves the unknown fields unset.

      • Resolved REST server issue

        Fixed problem where an enabled port would respond to HTTP requests when not desired.

      • Fixed issue with StatsD metrics collection

        Previous versions of the Sysdig agent, when configured to use the StatsD fowarder ({{statsd.use_forwarder: true}}) truncated messages that it received from containers to 2048 bytes, resulting in the potential for dropped and corrupted metrics. This change resolves that problem.

      0.90.3 June 14, 2019
      New Features

      StatsD port configurationThe Sysdig agent includes a configuration option, statsd.udp_port, that enables customers to select a non-standard port on which the agent should collect StatsD metrics.

      In previous releases of the agent, statsd.udp_port was respected for host metrics but not container metrics, when the statsd.use_forwarder option was not set to true.

      This change addresses that issue -- the agent will now respect the statsd.udp_port configuration both for the host and for all containers.

      See also: Integrate StatsD Metrics

      Fixed Issues
      Platform

      • App Check improvement

        Improved the integration (App Check) framework to detect mandatory config sections and generate errors when missing.

        This is currently enforced only for consumer groups required for the Kafka App Check.

      • Configurable timeout value for Prometheus metrics gatheringAdded a new Prometheus configuration option, timeout. Can be used to configure the amount of time the agent will wait while scraping a Prometheus endpoint before timing out. The default value is 1 second.See also: Integrate Prometheus Metrics.

      • eBPF Probe fix

        Fixed an issue in the eBPF-based Sysdig probe that was causing its build to fail in GKE COS versions 4.14.94 and higher.

      Monitor

      Metrics reporting fix

      Fixed an issue converting large 64-bit numbers, which could lead to loss of metrics such as memory.limit.bytes. This regression first occurred in agent version 0.90.0

      Secure

      Image scanning fix

      Fixed an issue that caused the agent not to report container image digest if the image were locally tagged.

      0.90.2 May 29, 2019
      New Features

      No new features in this release.

      Fixed Issues
      Platform

      • Reverted Kubernetes updates interval

        In agent release 0.90.1, the frequency of Kubernetes updates emitted from the agent to the backend was changed from 1 second to 4 seconds by default.This affects viewing samples at 1-second granularity when they are segmented by Kubernetes tags. The change of frequency also broke the calculation of rate aggregations for Kubernetes metrics.

        This fix reverts the default frequency back to 1 second.

      • Sysdig Inspect fixFixed reporting of container name and container image in Docker events when viewed in Sysdig Inspect.

      • Unhandled exception fixFixed an issue that resulted in an unhandled exception, which caused the agent to crash. The exception could be generated when dealing with unexpected values in internal events generated by the agent.

      • Linux 5.1.x supportMade changes to support Linux 5.1.x kernel versions.

      • Smart container filtering fix

        Container deletion during smart container filtering could cause the agent to crash. This fix alleviates that issue.

      • Unhandled object type fixFixed an issue that resulted in the crash of the cointerface process, due to an unhandled object type.

        The unhandled object is generated by the Kubernetes client-go library, in rare circumstances when the agent loses connectivity with the Kubernetes API server.

      • Docker Swarm task fix

        Fixed an issue that resulted in the crash of the cointerface process while processing a Docker Swarm task that does not have any containers.

      0.90.1 May 22, 2019
      New Features
      Monitor

      • Jenkins app check has been enhanced to report jobs in sub-folders of the default Jenkins home directory

        See Jenkins Folders Plugin for details.

      • Varnish metric handling changed to avoid clutter created by generating metric names with timestamps

        VBE metrics generated by Varnish app check are filtered out by default. If necessary, these metrics can be gathered by explicitly including them in dragent.yaml.

        See Varnishdocumentation for details.

      • The kafka_consumer app check has been enhanced to support authentication

        See Apache Kafka documentation for details.

      Fixed Issues
      Platform

      • Metadata bug fix

        Fixed an issue that resulted in the loss of a container's metadata when it was larger than 4kB.

      • Fix to make agent - backend connection more robust

        Fixed an issue that occurs when the agent is reconnecting to the collector and then gets restarted.

      • Provide StatsD option to run statsite_forwarder in driver mode

        The Sysdig agent automatically parses and records StatsD metrics. Historically, the agent parsed the system call stream from the kernel in order to read and record StatsD metrics from containers.

        For performance reasons, the agent may not be able to collect all StatsD metrics using this mechanism.

        (For example, if the StatsD client writes more than 2kB worth of StatsD metrics in a single system call, the agent will truncate the StatsD message, resulting in loss of StatsD metrics.)

        This change introduces a new configuration option:

        statsd:
  use_forwarder: true

        When set, this option enables the agent reliably to receive StatsD metrics from containers, even while the agent is under load.

        See Integrate StatsD Metrics for more detail.

      • net.bytes.* metric bug fix

        Fixed an issue with the calculation of net.bytes.* metrics under rare conditions when the agent sub-samples Sysdig events.

        Specified a port range for increased snapshot length (snaplen) in the default.dragent.yaml, to work around a StatsD truncation issue.

      • Agent crash issue

        A bug was causing the agent to crash when restarting if the connection to the backend failed or timed out. This issue has been resolved.

      • Activity timestampsImproved process watchdog to handle potential activity timestamps greater than the current time.

      • App check name in logsAdded app check name to log messages.

      • Reduce overhead for serializationReduced overhead of serializing messages sent to the Sysdig backend, including doing the serialization asynchronously.

      • Disallow running some app check integrations with Python 2.6

        Will not run the following app checks when using Python 2.6:

        NGINX Plus, HAProxy, Consul, Kafka, ElasticSearch, RabbitMQ

      • SSL certs for RHEL/CentOS

        Now uses the correct SSL CA bundle on RHEL/CentOS systems.

      • Asynchronous fetches Agent performs fetches of container metadata asynchronously.

      • Cointerface handling

        Gracefully reconnects to the cointerface process when it is restarted, by having a single gRPC connection.

        Increased cointerface's default watchdog limit to 1GiB.

      • Segmentation improvements

        Segments file I/O per file name for programs/containers, account file I/O per superblock.

      • Improved Kubernetes node reporting

        Sysdig Monitor now uses container information instead of IP address information to identify the Kubernetes node. This prevents reporting information for multiple Kubernetes nodes.

      • PV Kube state metrics improvement

        Ensures that Kube State Metrics are still collected after creating a Persistent Volume.

      • Event processing improvement

        Improved performance of system call event processing.

      • NULL file handling

        Fixed a potential crash when extracting file information for NULL files.

      Secure

      • Docker compliance report

        When running the Docker compliance check, limits the number of reported images/containers/etc. to 500.

      • Runtime policy at startup

        Ensures runtime policy scopes can use host-level information immediately at startup, rather than waiting for messages from the backend.

      • Empty schedule handling

        Handles the case in which the agent receives an empty schedule of compliance tasks

      0.88.0 January 28, 2019
      Enhancements

      • Added support for generating pre-built sysdigcloud-probes for Fedora Atomic kernels.

      • Optimized inter-process communication among Kubernetes subsystems of the agent to improve agent performance.

      • Optimized the processing of updates from the Kubernetes API server, resulting in significant CPU and memory consumption savings.

      Sysdig Secure

      • Added support for Kubernetes version 1.11 for kube-bench compliance checks.

      • Added a failure log when compliance tasks fail. Details on the failure are sent to the backend, and displayed in the Sysdig Secure UI.

      Fixed Issues
      Posix Queue Errors

      The Sysdig agent uses an inter-process communication (IPC) mechanism called POSIX queues to gather data from its subprocesses. An issue occurred when additional processes used these POSIX queues, as the configured resource limits were set for the Sysdig agentâs use only, resulting in the resource limits being reached and at times surpassed.

      To resolve the issue, the Sysdig agent is now always run in its own IPC namespace, isolating it from interference from other processes using POSIX queues. In addition, the queue limits can be configured manually in the dragent.yaml file if necessary.

      Note

      The queue limits cannot be set to less than the minimum Sysdig would automatically set them to.

      Backend Reporting Incorrect Kubernetes Grouping

      An issue occurred with the validation logic for parent-child matching between Kubernetes objects, resulting in the Sysdig backend reporting incorrect parents for Kubernetes. The validation logic has been fixed, and now clearly identifies parent components.

      JVM Metrics Not Reported

      An issue occurred that prevented the Java application from executing inside the container namespace of the target application when the default user and group of the application container did not have sufficient privileges. This issue has been resolved.

      0.87.1 January 10, 2019
      Enhancements
      Additional Internal Metrics

      The agent now generates two additional internal metrics:

      • dragent.analyzer.n_command_lines counts the number of command lines collected.

      • dragent.analyzer.n_container_healthcheck_command_lines counts the number of command lines collected that are associated with container health checks.

      Skip Health Check Command

      A new agent configuration option has been added to allow users to control whether command lines associated with the container health checks are collected and reported:

      commandlines_capture: {include_container_healthchecks: (true|false)}

      Note

      The default value is true, preserving existing behavior.

      Technical Previews / Beta Features
      Smart Container Filtering

      New sort logic has been implemented for environments that exceed the maximum number of containers that can be reported in each sample, to improve results. container_filter tags and container age are now taken into account, rather than solely basing the list on resource usage. This feature is disabled by default.

      Bug Fixes
      Invalid net.http.statuscode

      The net.http.statusCode metric was returning an invalid HTTP status code of -1, because of an issue with how the agent handled large strings. The logic has been updated, and the agent now collects the correct http status code.

      sdschecks no_proxy support

      In agent release 0.86.3, support was added for using the NO_PROXY environment variable in app checks. However, there was an error in the semantics, which prevented app checks from working when the variable was set. This has been corrected, and NO_PROXY now works as expected.

      Security Issues
      CVE-2018-7738: https://nvd.nist.gov/vuln/detail/CVE-2018-7738

      The bash-completion package was removed from the Sysdig Agent due to a potential security vulnerability.

      • Falco secure policies can now use the in operator with a set of addr/netmask values.

      • Added support for AMD's Secure Memory Encryption (SME).

      Bug fixes

      • Report negative offsets as zero in Kafka AppCheck.

      • Fixed hostname resolution in etcd AppCheck.

      • Bumped up agent's default memory limit for cointerface to 256 MiB.

      • Pruned list of known_ports in dragent.default.yaml for which net.request.* metrics are calculated.

      • Blacklisted DNS from known_ports to prevent DNS from consuming entries in agent's connection table.

      • Mark a protocol as TLS only if the port is in known_ports.

      • Output fields in secure policy events are specified in agent (and are more flexible).

      • Small fixes to filesystem-related secure policies.

      • Added /run/secrets to the mount points exclude list in dragent.default.yaml.

      0.78.1 Mar 15, 2018
      Bug Fixes

      • Fix a regression where some encryption certificates could not be parsed properly when accessing Kubernetes API.

      0.78.0 Mar 9, 2018
      New Features

      • Install script now works on Amazon Linux 2.

      • We have added a filter to prune most of the default labels that orchestrators add in order to reduce noise and redundant metadata. Container and Kubernetes labels are limitted to 50 chars max.

      Bug Fixes

      • UDP connection tracking improvements: a connection will be reported only if there has been at least one write/read between endpoints.

      • Fixed driver compilation on Ubuntu Xenial when using containerized agent.

      • Fixed a race condition that caused crashes of Kubernetes cointerface.

      • Fixed a bug where the agent would stop collecting metadata after a Kubernetes cointerface crash.

      • Avoid retrying the Kubernetes API if doesn't have CronJobs support when collecting state for those.

      • Fixed a bug that caused sdjagent (Java metrics collector) to stall.

      0.77.0 Feb 9, 2018
      New Features

      • Support for new Sysdig Secure policy types in addition to Falco based policies. Requires additional configuration on the backend side at the account level to enable them.

      • Add ability to scrape Prometheus histogram metric type.

      • AppChecks can now ignore all SSL warnings.

      Bug Fixes

      • Fix Gunicorn AppCheck compatibility with a more recent Python psutil package.

      • Fix Kafka AppCheck to handle cases when node_id is 0.

      • Improve accuracy of CPU usage calculations for newly started containers with recycled container IDs.

      • Add additional validation of and warnings about degenerate Mesos task IDs such as 0 and 1.

      • More aggressively filter container mounts to exclude common mounts set up by Docker and Kubernetes.

      • Fix regression in reporting container counts caused by not reporting CPU usage when 0.

      0.76.4 Jan 31, 2018
      Bug Fixes

      • Fixed calculation of CPU usage after missed samples.

      • Stuck captures no longer prevent the start of other captures.

      • Fixde a problem that could cause repeated events in captures.

      0.76.3 Jan 26, 2018
      New Features

      • Added ability to get Mesos lightweight metadata from Docker environment data if container processes zero out environment variables.

      Bug Fixes

      • Suppress SSL warnings from http_check AppCheck if it is configured to ignore warnings.

      • Fixed a rare crash that can occur during agent startup due to uninitialized internal state.

      • Changed level of Kubernetes delegated node log message from DEBUG to INFO.

      • Clamp CPU values when incorrect values are detected due to issues in kernels 4.8 - 4.10.

      0.76.2 Jan 18, 2018
      New Features

      • Support for JMX metrics from Java 9 applications.

      Bug Fixes

      • Fixed an issue in 0.76.1 that causes agent captures to not work with the Sysdig command line tool and Sysdig Inspect.

      • Added logic to handle invalid values in cgroup's cpuacct.usage.

      0.76.1 Jan 17, 2018
      New Features

      • Added Amazon Linux 2 support.

      • Dump make.log when dkms probe module builds fail.

      Bug Fixes

      • Fixed crash in setsid() exit event parser.

      • Upgraded various build dependencies.

      • Fix slow processing of cointerface RPC responses that can cause memory pressure.

      0.76 Jan 12, 2018
      New Features

      • Agent now reads container cpu.used.percent directly from cgroups instead of summing all the processes, so it will be more accurate.

      • There is a new command on sdjagent: allAvailableBeans that prints all the beans found on a JVM with their respective values.

      • Kubernetes metadata is available also for init containers.

      • Kakfa AppCheck support for consumer offsets stored in Kafka.

      Bug Fixes

      • sdjagent availableMetrics command now uses the same logic of getMetrics to detect JMX attributes convertible to numbers.

      • Fix a crash occurring if all statsd metrics are filtered.

      • Fixed an error when calculating the hash for precompiled CoreOS probe modules.

      • Fixed an agent probe module failure on Ubuntu kernels with the Meltdown patches that disable the page-fault tracepoints.
